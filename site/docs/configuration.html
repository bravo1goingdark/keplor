Configuration

TOML config file with environment variable overrides.

Config file

By default, Keplor looks for keplor.toml in the working directory. Override with --config:

$ keplor run --config /etc/keplor/keplor.toml

Full example

# keplor.toml

[server]
listen_addr = "0.0.0.0:8080"
shutdown_timeout_secs = 25

[storage]
db_path = "/var/lib/keplor/keplor.db"

[auth]
api_keys = ["sk-keplor-prod-abc123", "sk-keplor-prod-def456"]

[pipeline]
batch_size = 128
max_body_bytes = 10485760

Reference

[server]

KeyTypeDefaultDescription
listen_addrstring"0.0.0.0:8080"Address and port to bind the HTTP server
shutdown_timeout_secsu6425Seconds to wait for in-flight requests during graceful shutdown

[storage]

KeyTypeDefaultDescription
db_pathstring"keplor.db"Path to the SQLite database file. Created if it doesn't exist.

[auth]

KeyTypeDefaultDescription
api_keysstring[][]List of valid API keys. When empty, authentication is disabled (open access). A warning is logged at startup.

[pipeline]

KeyTypeDefaultConstraintsDescription
batch_sizeusize641 – 100,000Max events accumulated before forcing a batch flush
max_body_bytesusize104857601 – 104,857,600Max HTTP request body size in bytes (10 MB default)

Environment variables

All config keys can be overridden with environment variables using the KEPLOR_ prefix and underscore nesting:

$ KEPLOR_SERVER_LISTEN_ADDR=0.0.0.0:9090 \
  KEPLOR_STORAGE_DB_PATH=/tmp/keplor.db \
  KEPLOR_AUTH_API_KEYS=sk-key1,sk-key2 \
  keplor run

Environment variables take precedence over the config file.

SQLite pragmas

Keplor applies these SQLite pragmas at startup for performance:

PragmaValuePurpose
journal_modeWALWrite-ahead logging for concurrent reads
synchronousNORMALFsync on checkpoint only, not every commit
mmap_size256 MBMemory-mapped I/O for read performance
busy_timeout5000 msRetry on lock contention
cache_size64 MBPage cache for query performance
temp_storeMEMORYTemp tables in RAM

Validation

Config values are validated at startup. Invalid values cause an immediate error with a descriptive message:

Error: invalid config: pipeline.batch_size must be > 0
