Quickstart

From zero to ingesting events in under 2 minutes.

1. Install

Build from source with Cargo:

$ git clone https://github.com/bravo1goingdark/keplor.git
$ cd keplor
$ cargo build --release
$ cp target/release/keplor /usr/local/bin/

Or build a static musl binary for deployment:

$ cargo build --release --target x86_64-unknown-linux-musl

2. Start the server

$ keplor run

Keplor starts on 0.0.0.0:8080 with an in-directory keplor.db SQLite database. No configuration file needed for local development.

Verify it's running:

$ curl http://localhost:8080/health
{"status":"ok","version":"0.1.0","db":"connected"}

3. Send your first event

$ curl -X POST http://localhost:8080/v1/events \
  -H "Content-Type: application/json" \
  -d '{
    "model": "gpt-4o",
    "provider": "openai",
    "usage": {
      "input_tokens": 500,
      "output_tokens": 200
    },
    "latency": {
      "total_ms": 1200,
      "ttft_ms": 180
    },
    "user_id": "alice",
    "source": "my-app"
  }'

Response:

{
  "id": "01J5XQKR...",
  "cost_nanodollars": 6250000,
  "model": "gpt-4o",
  "provider": "openai"
}

The cost is automatically computed from the LiteLLM pricing catalog. 6250000 nanodollars = $0.00625.

4. Query it back

$ curl "http://localhost:8080/v1/events?user_id=alice&limit=5"

Or use the CLI:

$ keplor query --user-id alice
ID                           PROVIDER       MODEL                   TOKENS       COST ($)
--------------------------------------------------------------------------------------------
01J5XQKR...                  openai         gpt-4o                     700     0.00625000

1 event(s)

5. Check storage stats

$ keplor stats
=== Keplor Storage Statistics ===
Database:             keplor.db
Total events:         1
Unique blobs:         2
Compressed size:      156 bytes
Uncompressed size:    312 bytes
Compression ratio:    2.0x
